# 会员页面设计说明

- 页面：会员
- 路由：src/pages/member

- 实现文件：
  - src/pages/member/index.tsx
  - src/pages/member/index.scss
  - src/pages/member/index.config.ts

## 设计意图

主包会员聚合入口，串联会员中心和权益能力。

## 页面结构

- 页面根容器：`_pg`
- 页面容器：`PageShell`
- 页面运行时：`usePageRuntime`
- 页面状态订阅：`observer`
- 内容区域：
  - 会员身份卡：昵称、等级、积分、卡券和会员码入口
  - 会员基础入口：会员码、优惠券、订单、地址、售后、分享乐园
  - 会员权益卡：生日礼遇、商城会员价、优先预定

## 动态与静态边界

- 页面图片：当前页面无真实图片区域。
- 接口数据：读取全局会员态，不直接请求接口。
- 本地配置：页面标题、导航策略、路由和分包注册。

## 状态要求

- loading：页面运行时统一承接。
- empty：后续优先使用 `BaseEmpty`。
- error：后续优先使用 `BaseException` 或 `StatusException`。
- 未登录：需要身份时使用 `usePageRuntime({ loginRequired: true })` 或 `AuthAction`。

## 接口与 Service

| 模块 | service | 失败策略 | 是否阻断页面 |
|---|---|---|---|
| 会员态摘要 | `rootStore.member` | 全局状态兜底游客态 | 否 |

## 交互与跳转

- 会员中心按钮：登录后进入 `src/pkg-member/pages/index`。
- 积分、卡券、会员码统计：登录后分别进入会员中心、优惠券、会员码。
- 会员基础入口：登录后进入会员码、优惠券、订单、地址、售后；分享入口调用微信分享菜单。
- 会员权益卡：登录后进入优惠券、商城首页或门票预定。

## 交互矩阵

| 元素 | 处理结果 |
|---|---|
| 立即登录 / 会员中心 | 登录后跳会员中心 |
| 乐园积分 | 登录后跳会员中心 |
| 可用卡券 | 登录后跳优惠券 |
| 会员核销 | 登录后跳会员码 |
| 会员码 / 优惠券 / 我的订单 / 地址管理 / 售后记录 | 登录后跳对应页面 |
| 分享乐园 | 登录后打开微信分享引导 |
| 生日月礼遇 / 商城会员价 / 优先预定 | 登录后跳优惠券、商城或门票预定 |

## 状态矩阵

| 状态 | 页面表现 |
|---|---|
| 未登录 | 显示游客态和 `--` 统计，所有敏感入口走登录守卫 |
| 已登录 | 展示会员昵称、等级、积分和可用卡券 |
| 分享菜单不可用 | 由 `showWechatShareGuide` 降级为 toast 提示右上角分享 |

## 微信开发工具验收清单

- 未登录点击会员中心、积分、卡券、会员码、入口卡：应先弹登录。
- 登录后点击会员码、优惠券、订单、地址、售后：应进入对应页面。
- 点击分享乐园：应出现微信分享引导提示。
- 点击三个权益卡：应分别进入优惠券、商城首页、门票预定。
