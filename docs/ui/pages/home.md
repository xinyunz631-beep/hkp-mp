## 设计意图

首页是乐园 C 端服务聚合入口。页面以固定顶部搜索栏、顶部运营 banner 和会员卡承接首屏识别，再通过营业时间、榜单、推荐和分类模块组织内容；原先的交通指南 / 乐园导览按钮已在当前首屏稿中先隐藏。动态图片不在设计稿中还原真实素材，代码侧统一交给 `AppImage` 承接灰色占位、加载中、失败态和成功淡入。

本页当前以 Pencil `CJZyH` 作为主设计事实源；Figma `LCJbfpQmHoCR5RdKTYcf0O` 保留为可切换参考来源。后续设计、调整和锁定都以 `page-registry.yaml` 的 `currentTool` 为准，开发前也要按这个字段去对应客户端看图。

当前首页已进入 `v1.15-implementing`：当前开发稿基于 `image1-3` 现网首页分屏重新拆分模块结构，并继续按用户最新反馈收紧首屏层级。顶部导航保持透明浮层，banner 直接露在其下方；快捷入口改成图标整块表达，不再显式展示入口文案；开园时间卡已改成参考图那种上图下文的整卡结构，保留大图占位和底部白色信息条；底部 tabbar 维持五槽位浮起结构，中央会员码只保留图标不再显示文字。首页首屏高还原 review 节点仍用于对齐用户提供的图 2，专门核对顶部导航、会员卡、开园时间卡和底部 tabbar 的首屏结构关系。当前首页 render 和 SCSS 已统一迁到 `_pg-*` 单下划线 BEM，页面约束检查也会拦截页面名前缀和双下划线元素写法。

5. 营业时间卡片

9. 玩转乐园九宫格
10. 页面内固定 tabbar

## 动态与静态边界

- 接口图片：顶部 banner、快捷入口整图、营业时间横图、榜单卡图、中部活动横幅、精彩推荐卡图、会员专享福利图、玩转乐园分类图，当前均以 render 内空字符串变量接入 `AppImage`，并通过 `emptyState="error"` 显示空地址失败态。
- 固定图片占位：未接入真实地址前由 `AppImage` 渲染灰色背景，不在页面内手写图片 icon 或技术文案。
- 接口文本/数据：会员昵称、会员等级、积分、优惠券数量、榜单和推荐模块标题/说明。
- 代码渲染：页面框架、搜索栏、会员卡、快捷入口、模块标题、卡片容器、标签、tabbar。
- 本地配置：当前快捷入口、分类标签和部分占位文案可先由页面配置数组承载，后续按接口能力替换。

## 状态要求

- loading：首页使用 `usePageRuntime({ initPage })` 管理首屏初始化。
- empty：运营内容为空时保留模块标题，可展示统一空状态或隐藏列表，具体随接口设计补充。
- error：优惠券数量失败不进入首页错误态。
- 未登录：签到、会员类快捷入口和会员福利入口触发登录弹窗。
- 降级态：优惠券数量接口失败由 service 返回 `0`。

## 接口与 Service

| 模块 | service | 失败策略 | 是否阻断页面 |
|---|---|---|---|
| 优惠券数量 | `fetchCouponUsedCount` | 方法内部 catch，返回 `0` | 否 |

## 交互与跳转

- 顶部扫码：调用微信 `scanCode`；扫码结果是分包路径时直接跳转，否则弹微信 modal 并支持复制。
- 顶部搜索：跳转商城搜索页 `src/pkg-mall/pages/search`。
- 首页 banner：主推门票预定、票务首页、会员中心；会员内容先走登录守卫。
- 快捷入口：兑换专区、领券中心、服务专区、官方商城、联系客服、分享收益、导航至乐园、园内地图均有登录守卫、微信 API 或分包跳转结果。
- 今日开园时间：跳转票务首页，承接节目单和票务服务。
- 热玩榜单、精选活动、精彩推荐、玩转乐园：全部可点击，按配置跳转票务、酒店、商城、地图、电话或暂缓提示。
- 会员福利卡：登录后进入会员中心。

## 交互矩阵

| 元素 | 处理结果 |
|---|---|
| 扫码按钮 | 微信扫码；结果可跳转或复制 |
| 搜索框 | 跳商城搜索 |
| banner 轮播项 | 跳门票预定 / 票务首页 / 登录后会员中心 |
| 8 个快捷入口 | 对应分包页面、微信电话、微信地图、登录守卫或分销暂缓提示 |
| 开园时间卡 | 跳票务首页 |
| 查看全部 | 按板块跳票务首页或园区导览 |
| 榜单 / 活动 / 推荐卡 | 跳详情、导览或微信地图 |
| 会员福利卡 | 登录后跳会员中心 |
| 玩转乐园九宫格 | 跳酒店、票务、商城、导览、微信地图、电话或餐饮暂缓提示 |

## 状态矩阵

| 状态 | 页面表现 |
|---|---|
| loading | `usePageRuntime` 承接首页初始化 |
| 未登录 | 会员相关动作调用登录弹窗，登录后续执行 |
| 扫码取消 | 微信 toast 提示未完成扫码，不阻断页面 |
| 地图失败 | 自动复制乐园地址作为降级 |
| 拨号失败 | 自动复制客服电话作为降级 |
| 分销 / 餐饮暂缓 | 保留计划内暂缓提示，不进入空白页 |

## 微信开发工具验收清单

- 进入首页，点击顶部扫码：应拉起微信扫码；取消后页面不异常。
- 点击搜索框：应进入商城搜索页。
- 点击 banner、热玩榜单、精选活动、精彩推荐、玩转乐园卡片：应有跳转、地图、电话或暂缓提示。
- 未登录点击兑换专区、领券中心、会员福利：应先弹登录，登录后继续跳转。
- 点击联系客服、导航至乐园：应优先走微信电话和微信地图能力。
