# 页面状态规范

所有页面设计和开发都必须考虑状态，不只做正常态。

## 必备状态

- 正常态：接口成功或默认数据可展示。
- 首屏 loading：首屏必须等待的初始化接口。
- 局部 loading：按钮提交、局部刷新、二次加载。
- 空状态：列表、订单、卡券、地址等无数据。
- 错误态：必须阻断页面的接口失败。
- 未登录态：需要身份的页面或入口。
- 降级态：可选接口失败但页面仍可继续。

## 设计要求

- loading、empty、error 使用统一母版，不为每页发明新视觉。
- 可降级接口失败不展示技术错误文案。
- 用户可见文案必须是业务口径，禁止出现 `mock`、`Taro`、`V2`、组件库、开发态、测试态等字眼。
- 状态稿要写进页面文档，说明触发条件和代码组件。
- 空状态默认使用浅灰占位图形 + 主文案 + 轻说明，对应 `BaseEmpty`。
- 错误态默认使用浅粉承托，不做重色全屏惊吓式反馈，对应 `BaseException`。
- 未登录态优先使用白卡 + 说明 + 主按钮的温和引导，登录动作由 `LoginPopup` 承接。
- 营销图、榜单图、分类图在真实接口返回前统一使用 `接口图占位`，不直接复刻截图素材。

## 代码要求

- 首屏依赖初始化接口时用 `usePageRuntime({ initPage })` + `renderPage`。
- 可降级接口由 service 返回默认值，不把异常抛给页面失败态。
- 必须阻断的接口才允许进入页面错误态。
